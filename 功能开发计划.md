# 功能开发计划(欠缺功能路线图)

> 本文档独立于 `开发计划.md`(后者记录已实现模块的状态与历史修复),专门梳理**当前尚缺、面向"数据采集工具"核心价值的功能缺口**,并排出开发优先级,供持续演进。
>
> 状态图例:📋 待规划 ｜ 🔄 进行中 ｜ ✅ 已完成 ｜ ⚠️ 部分实现/有盲点
>
> 工作量:S(小,数十行~1天) ｜ M(中,2~4天) ｜ L(大,1周+)
>
> 生成日期:2026-07-06。基于对 `src/core`、`src/renderer`、`src/main` 六个维度的只读代码调研。

---

## 一、现状定位

这是一个**录制式网页数据采集器**,在"单次、单参数、手动触发"的采集上已相当成熟:

**已具备**:10 种步骤类型(goto/click/fill/press/scroll/scroll-bottom/wait-for-load/waitForSelector/waitForClickable/pause)、4 种提取模式(single/list/list-detail/list-action)、翻页鲁棒性、选择器指纹自愈、AI 生成/校正选择器、下载捕获、Excel 导出、后处理插件、反检测(抹 webdriver + 真内核回退)、登录态复用、暂停/停止/撤销重做。

**功能天花板卡在三个方向**:**① 录不下来**(交互覆盖不全)、**② 采了不好用**(数据加工几乎为零)、**③ 规模化不了**(无参数化/定时/重试/断点续采)。下面按价值杠杆排出路线图。

---

## 二、缺口全景(六大能力域)

| 能力域 | 现状 | 主要缺口 |
| -- | -- | -- |
| A. 交互录制与回放 | 仅录 点击/输入/7 个特殊键/滚动 | select 下拉、checkbox/radio、hover、文件上传、双击、右键、拖拽、快捷键组合、iframe 内元素 |
| B. 数据加工与导出 | 仅 `trim()`;仅导出 xlsx | 无正则/替换/格式化/拼接、无去重、无空行过滤;无 CSV/JSON;详情页无多层嵌套 |
| C. 自动化与编排 | 手动点"运行"、fill 值写死 | 无参数化(数据驱动)、无定时、无变量传值、无条件/循环、无工作流、无并发、无 CLI |
| D. 反爬与访问控制 | 基础反检测 | 无代理/IP 轮换、无限速/随机延时、无 UA/指纹轮换、无验证码方案、无多账号池 |
| E. 可靠性与可观测 | 失败即中止、全内存 | 无重试、容错不全、无进度/ETA、无完成/失败告警、无运行历史、无断点续采、无流式落盘 |
| F. 宏资产管理 | 靠文件对话框存取 | 无宏库列表 UI、元数据极简、无版本历史、无模板库、无导入导出、无项目分组 |

## 三、开发路线图

### 🥇 第一梯队(P0):把工具从"能用"推到"好用"

> 这些是回报最高、相对独立、改动可控的增量,直接决定核心采集价值。

#### P0-1 参数化 / 数据驱动 📋 · 工作量 M
- **现状**:`fill.value` 是录制时写死的固定文本([macro-types.ts](src/core/macro-types.ts) FillStep),`goto.url` 同样固定。一个"搜索'手机'"的宏永远只能搜"手机"。
- **缺口**:无 `${变量}` 占位符;无"喂一个关键词/URL/账号列表(CSV/JSON),同一宏跑 N 轮、每轮换参数"的批量能力。
- **价值**:**最高杠杆**。把工具从"录一次跑一次的个人玩具"升级成"一个模板批量采集"。
- **方案要点**:① 步骤字段(fill.value / goto.url / 可扩展至 selector)支持 `${col}` 占位符;② 运行入口支持导入数据集(每行一组变量);③ 回放主循环外层套数据集迭代,每轮注入变量、结果按行/按数据集分文件落盘。
- **涉及**:`macro-types.ts`(Macro 加 variables/dataset 概念)、`macro-runner.ts`(注入层)、`renderer.ts` + `index.html`(数据集导入 UI)。

#### P0-2 补齐高频交互录制:select / checkbox / hover 📋 · 工作量 M
- **现状**:录制端只监听 点击、input、7 个特殊键、滚动([webview-preload.ts](src/renderer/webview-preload.ts))。下拉框/勾选/悬停用户做了却录不进。
- **缺口**:① `<select>` 没监听 `change`,回放只点一下不知选哪项;② checkbox/radio 只录了点击没录选中态;③ hover 菜单完全没监听 `mouseover`。
- **价值**:筛选类页面(地区/分类/排序)的标配交互,一旦用到当前就录不全。回放引擎(Playwright `selectOption`)本身够用,缺的是录制端事件 + 步骤类型。
- **方案要点**:新增步骤类型 `select`(记 `{selector,value}`)、`check`(记选中态)、`hover`;录制端补 `change`/`mouseover` 监听;回放端 `page.selectOption`/`setChecked`/`hover`。
- **涉及**:`macro-types.ts`(StepType 扩展)、`webview-preload.ts`(事件捕获)、`macro-runner.ts`(handle 分支)、`renderer.ts`(describeStep + 右键插入)。

#### P0-3 字段数据加工:transform + 去重 + 空行过滤 📋 · 工作量 M
- **现状**:提取只有一个 `.trim()`([extractor.ts:163](src/core/extractor.ts#L163)),之后原样吐出。
- **缺口**:① 无正则截取/替换(如 "¥1,299.00 元"→1299);② 无去重(翻页重复项全保留);③ 无空行过滤(全空字段行照样进表);④ 无格式化/多字段拼接。
- **价值**:用户现在拿到脏数据还得再洗一遍。纯 core 层改动、不碰 UI 交互。
- **方案要点**:`ExtractField` 加可选 `transform`(regex/replace/prefix/suffix/trim 组合)、`ExtractConfig` 加 `dedupe`(按指定列或整行)与 `skipEmptyRows`;在 extractor 收集处统一应用。
- **涉及**:`macro-types.ts`、`extractor.ts`。

#### P0-4 失败重试 + 增量/流式落盘 📋 · 工作量 M
- **现状**:**0 处重试**,第一个错误即进 catch 返回失败([macro-runner.ts](src/core/macro-runner.ts) 主循环);提取全程堆内存,采完才返回([extractor.ts:127-134](src/core/extractor.ts#L127)),进程崩溃/强杀已采数据全丢。
- **缺口**:无整宏/单步/失败项重试;无边采边落盘。
- **价值**:无人值守与大任务的**可靠性基础**。采到 80% 崩溃不该丢掉前 80 万行。
- **方案要点**:① `Macro` 加 `retry:{maxRetries,retryDelay,backoff}`,导航/点击等关键步骤失败按指数退避重试;② 提取每页/每 N 行流式追加写 CSV(或临时 JSONL),完成再合并导出,支持崩溃后保留。
- **涉及**:`macro-types.ts`、`macro-runner.ts`、`extractor.ts`、新增 streaming 落盘工具。

#### P0-5 请求限速 / 随机延时 📋 · 工作量 S
- **现状**:除 list-action 逐项等下载外,无访问频率控制;仅有硬编码 `waitForTimeout(1000)`。
- **缺口**:无全局步骤间延时、无翻页间隔、无随机抖动。
- **价值**:**最低成本的反爬**,规律高频访问最易触发封禁/限流。
- **方案要点**:`SessionOptions` 加 `stepDelayMs` + `randomJitterMs`,在 `executeStep` 与翻页之间插入延时(带随机);UI 一个"访问间隔"输入。
- **涉及**:`macro-types.ts`、`macro-runner.ts`、`extractor.ts`(翻页处)、`renderer.ts`。

#### P0-6 多格式导出:CSV / JSON 📋 · 工作量 S
- **现状**:仅 xlsx([excel-exporter.ts](src/core/excel-exporter.ts))。
- **缺口**:无 CSV/JSON——恰是喂下游程序/数据库/BI 最常用格式。
- **价值**:rows 已是结构化数组,成本极低,顺手做。
- **方案要点**:导出器抽象出 `format` 参数,补 CSV(UTF-8 BOM,兼容 Excel 打开)与 JSON;UI 导出按钮加格式选择。
- **涉及**:`excel-exporter.ts`(或新 exporter 模块)、`main.ts` IPC、`renderer.ts`。

---

### 🥈 第二梯队(P1):迈向"可复用、能托管"

#### P1-1 定时 / 计划任务 📋 · 工作量 M
- **现状**:只能手动点"运行",无 cron。
- **方案**:集成 node-schedule/cron,支持"每天/每小时/cron 表达式"定时跑指定宏,后台运行 + 任务列表管理。与 P0-1 参数化叠加即"每天定时批量采"。
- **涉及**:`main.ts`(调度器)、`renderer.ts` + `index.html`(计划任务面板)。

#### P1-2 变量与步骤间传值 📋 · 工作量 M
- **现状**:提取到的值无法回喂后续步骤;list-detail 的"取链接→进详情"是硬编码模式,通用传值做不了。
- **方案**:引入运行期变量表,步骤可"抓取某元素文本/属性存入变量",后续 fill/goto 用 `${var}` 引用。为 P2 的条件/循环打基础。
- **涉及**:`macro-types.ts`、`macro-runner.ts`。

#### P1-3 代理 / IP 配置 📋 · 工作量 M
- **现状**:Playwright launch 未用 proxy 选项,`BrowserConfig`/`SessionOptions` 无代理字段。
- **方案**:支持 HTTP/SOCKS 代理(含认证);进一步支持代理列表轮换(每轮/每次回放换 IP)。
- **涉及**:`macro-types.ts`、`macro-runner.ts`(launch/context proxy)、`browser-config.json` + 登录态面板 UI。

#### P1-4 宏库管理 UI + 宏元数据 📋 · 工作量 M
- **现状**:每次靠文件对话框选文件,`macrosDir` 未在 UI 呈现;`Macro` 只有 name/version/steps/extract/postProcess([macro-types.ts](src/core/macro-types.ts) Macro)。
- **方案**:① 侧栏"宏库"列表(列出 macros/ 下所有宏,支持搜索/重命名/复制/删除/一键打开);② `Macro` 扩展 `description/targetSite/tags/createdAt/updatedAt/note`,加宏信息编辑面板。
- **涉及**:`macro-store.ts`(list 接口)、`main.ts`(IPC)、`macro-types.ts`、`renderer.ts` + `index.html`。

#### P1-5 运行进度 / ETA + 完成/失败告警 📋 · 工作量 M
- **现状**:长任务只有文本日志,唯一进度是"第 p/N 页累计 X 行";无外发通知。
- **方案**:① 结构化进度事件(当前页/行数/吞吐/ETA)推到 UI 进度条;② 完成/失败经 桌面通知 + 可选 webhook(钉钉/企业微信/Slack)推送结果摘要。
- **涉及**:`extractor.ts`(发 onProgress)、`macro-runner.ts`、`main.ts`、`renderer.ts`。

#### P1-6 更多交互:文件上传 / 双击 / 快捷键组合 📋 · 工作量 M
- **现状**:input[type=file] 的 change、dblclick、Ctrl/Shift 组合键均未录制;press 只认 7 个键。
- **方案**:新增 `upload`/`dblclick` 步骤类型 + 录制监听;press 扩展支持修饰键组合(回放端 Playwright 已支持)。
- **涉及**:`macro-types.ts`、`webview-preload.ts`、`macro-runner.ts`、`renderer.ts`。

#### P1-7 宏克隆 / 导入导出 📋 · 工作量 S
- **现状**:无"导出宏"按钮(只能手动复制 json),loadMacro 靠对话框。
- **方案**:导出当前宏为 .json 或含旁车的 .zip;拖拽/文件导入;"复制此宏"派生副本。
- **涉及**:`main.ts`、`macro-store.ts`、`renderer.ts`。

### 🥉 第三梯队(P2):走向"自动化平台"与"高防护站点"

> 复杂度递增,按业务需要选做。

#### P2-1 条件分支 / 通用循环 📋 · 工作量 L
- 无 `if 元素存在则…`、无翻页以外的循环。补条件步骤与循环步骤,应对页面结构不一致与错误恢复。依赖 P1-2 变量。
- 涉及:`macro-types.ts`、`macro-runner.ts`(执行引擎需从线性升级为可跳转)。

#### P2-2 多宏工作流 / 串联 📋 · 工作量 L
- 无法"登录宏 → 采集宏 → 合并宏"串联,无宏调宏。设计工作流 DSL,支持有序执行 + 宏间结果传递。
- 涉及:新增 workflow 层、`main.ts`、UI。

#### P2-3 运行历史库 + 结果对比 📋 · 工作量 M
- 每次运行结果分散落盘,无统一历史。引入 SQLite 记录每次运行(时间/宏/行数/耗时/错误),支持"成功率、耗时趋势、高频错误排行、上次 vs 本次数据对比"。
- 涉及:新增 `src/storage/run-history.ts`、UI 历史面板。

#### P2-4 验证码方案 📋 · 工作量 M
- 目前仅 pause 人工介入。对接打码平台(2captcha 等)/OCR,半自动处理 reCAPTCHA/hCaptcha 与图形码。
- 涉及:新增 `captcha-solver.ts`、pause 流程集成。

#### P2-5 UA / 指纹轮换 + 多账号池 📋 · 工作量 M
- UA 固定、viewport/时区固定、无账号切换。补 UA/viewport/时区随机化池、Canvas/WebGL 指纹扰动、多账号 profile 池轮换。深防护站点(Cloudflare/DataDome)可评估切 patchright。
- 涉及:`macro-runner.ts`、`browser-config.json`、账号管理 UI。

#### P2-6 iframe 内元素录制 📋 · 工作量 L
- preload 只注入顶层文档,iframe 内交互(嵌入表单/编辑器/支付框)全录不到。需向子 frame 注入并跨 frame 定位。架构性改动。
- 涉及:`webview-preload.ts`、`macro-runner.ts`(frame 定位)。

#### P2-7 编辑体验增强:批量编辑 / 步骤复制 / JSON 直编 / 步骤模板 📋 · 工作量 M
- 现无批量删/复制步骤、无原始 JSON 编辑器、无常用步骤序列模板。逐项补齐,提升宏编辑效率。
- 涉及:`renderer.ts` + `index.html`。

#### P2-8 宏版本历史 + 项目分组 📋 · 工作量 M
- version 字段恒为 1、Undo/Redo 仅会话内、macros/ 扁平存放。补:保存自动版本化 + 回退;宏归入"项目"分组、支持子目录层级与项目级批量运行。
- 涉及:`macro-store.ts`、`macro-types.ts`、UI。

#### P2-9 并发 / CLI 化 📋 · 工作量 L
- 单窗单跑、无 CLI。core 层已不依赖 Electron(verify 脚本可证),可封装 `macro run xxx.json --headless` 命令 + 任务队列并发跑多宏,便于外部系统集成与规模化。
- 涉及:新增 CLI 入口、任务调度层。

#### P2-10 list-detail 多层嵌套 / 详情页分页 📋 · 工作量 M
- 详情提取只有两层(列表→详情),无法进详情页再进子详情,详情页内也无法翻页。按需扩展递归提取。
- 涉及:`extractor.ts`、`macro-types.ts`。

## 四、优先级速览

```
P0(先做,核心价值)
  1 参数化/数据驱动        —— 一个宏批量跑,杠杆最高
  2 select/checkbox/hover  —— 解锁一大批录不了的站
  3 字段 transform+去重     —— 数据直接可用
  4 失败重试+流式落盘       —— 无人值守可靠性基础
  5 请求限速/随机延时       —— 最低成本反爬
  6 CSV/JSON 导出          —— 顺手,喂下游

P1(可复用/能托管)
  定时任务 · 变量传值 · 代理配置 · 宏库UI+元数据 ·
  进度ETA+告警 · 文件上传/双击/组合键 · 宏克隆导入导出

P2(平台化/高防护)
  条件循环 · 工作流 · 运行历史库 · 验证码 · 指纹轮换/账号池 ·
  iframe · 编辑增强 · 版本/项目分组 · 并发/CLI · 详情多层嵌套
```

**建议**:P0 六项做完,工具即从"能用"跃升到"好用且可批量";其中 **P0-1 参数化** 若只做一件事,首选它。
