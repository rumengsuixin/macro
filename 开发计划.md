技术栈:Node + TypeScript + Electron + Playwright + exceljs,本地 JSON 存储。

## 模块:回放引擎(MacroRunner)

### 核心功能

- ✅ 读取 JSON 宏,Playwright 启动 Chromium(headed)逐步执行,每步中文日志,出错截图存 `errors/` + 结构化错误
- ✅ 每种 step 独立处理方法(goto/click/fill/press/scroll/scroll-bottom/wait-for-load/waitForSelector/waitForClickable/pause)
- ✅ 回放全局默认超时(默认 60s,`MACRO_TIMEOUT` 毫秒覆盖)
- ✅ 翻页步骤处理:正常回放跳过翻页步骤,收集构造「翻页序列」`turnPage`(翻页后 `waitForLoadState` 稳定),总页数取 `pageCount` 最大值驱动提取
- ✅ click 通用降级:`handleClick` 命中 ≠1 时用指纹 `relocateByFingerprint`(anchor→文本→aria→href),无指纹回退原生 click
- ✅ 人工介入暂停 `pause` 步骤(`onPause` 回调依赖倒置,`runId` 隔离 resume,可选 timeout)
- ✅ 运行中停止/中断回放(`cancel()` 置取消标志 + 关 `activeContext` 打断慢步骤,`RunResult.cancelled`)
- ✅ 滚动到底部 `scroll-bottom`(window + 所有内部可滚动容器滚到底 + 等 1s 懒加载)
- ✅ 等待页面加载完成 `wait-for-load`(`waitForLoadState('load')`,超时非致命)
- ✅ 等待元素可点击 `waitForClickable`(`waitForFunction` 纯只读判定尺寸/可见/非disabled/遮挡,不用 trial click 免副作用,超时致命)
- ✅ 登录态复用(三开关:持久化 profile / 注入录制 cookie / 注入录制 localStorage(限当前页 origin),`SessionOptions` 第 4 参)
- ✅ 回放时回填步骤来源 URL(`RunResult.stepUrls` 每步 `activePage.url()`,`backfillRecordedUrls` 只补缺失,救旧宏 URL 分组)
- ✅ 回放反检测加固(启动参数 + 内核回退链 `chrome`→`msedge`→捆绑 + `addInitScript` 抹 webdriver,`useSystemChrome` 默认开)
- ✅ 等待类步骤跟随迟到的新窗口(`raceWaitAcrossPopup` 当前页与新弹窗竞态等待)

## 模块:请求拦截改写(仅回放端 Playwright route/CDP;8 支路,配置 `request-rules.json`)

> 全字段速查见 `拦截器规则配置手册.html`;改写纯逻辑 `src/core/request-rewrite.ts`,回放端 `macro-runner.ts`,录制端 `request-interceptor.ts`(现已不接线、仅自检直用)。

### 核心功能

- ✅ 支路 `rules`:命中 POST 改 body 顶层字段(set→append→remove,数组值直接写真 JSON 数组、append 去重);`context.route` 全量拦截 + core `globToRegExp` 匹配,`route.continue({postData})`
- ✅ 支路 `resends`:命中后延时改参**主动重发新请求**(不改原请求),页面内 `activePage.evaluate(fetch)`(DevTools 可见 + 带登录态),防递归标记头 `x-macro-resend`,定时器 Set 清理
- ✅ resends 扩展:`setHeaders`/`removeHeaders` 改重发请求头、`replaceWithFile` 本地文件整体替换重发 body(发任意二进制)、`responseTrigger` 响应条件触发(捕获-重放模型,`triggerUrl` 必填,status/headers/requestHeaders/bodyJson/bodyContains 全 AND;`requestHeaders` 判 triggerUrl 请求侧的头)
- ✅ 响应触发**链式连环**:标记头 `x-macro-resend` 由「防递归布尔」升级为「链上跳数」(真实请求 hop0、每重发 +1),一条重发的响应可继续触发下一条规则(A→B→C);跳数达 `maxResendHops`(配置项,缺省 5、clamp[1,100])即熔断——支持连环的同时兜底防无限自环/互环
- ✅ 响应触发 `when` 表达式:`responseTrigger.when` JS 风格布尔表达式(零依赖迷你引擎 `expr-eval.ts`:tokenizer+Pratt+安全求值器,禁原型逃逸/全局),与静态条件 AND;上下文 `status`/`hop`/`body`/`text` + `header()`/`reqHeader()`/`match()`/`contains()`;核心 `hop == 0` 只在真实响应触发、从根切断相互连环(不再依赖 maxResendHops 兜底);失败即安全(解析/求值失败判不命中)
- ✅ 响应触发**取值注入**:`responseTrigger.extract`(变量名→`fromBody` 响应体点路径 / `fromHeader` 响应头名,取不到走 `default`)从触发响应提取命名变量,动作字段 `set`/`append`/`setHeaders`/`setUrl` 值里用 `{{变量}}` 占位注入重发(请求头仅 set;`set`/`append` 深度渲染字符串叶子;`setUrl` 整体覆盖重发目标 URL、渲染为空回退原 URL);纯逻辑 `extractResendVars`/`renderTemplate`/`renderResendActions`,`triggerNeedsBody` 遇 `fromBody` 亦读体
- ✅ 支路 `responseRules`:响应头满足 `when`(AND、大小写不敏感)时 `setHeaders`/`removeHeaders`(`route.fetch`+`route.fulfill`,可与 body 改写组合)
- ✅ 支路 `requestHeaderRules`:命中且原始请求头满足 `when` 时改**原始请求头**(`content-length` 自动剥离;cookie/host 改不了);route handler 三组合协同(仅头/头+body/头+响应头)一次 continue 合并
- ✅ 支路 `blocks`:命中即硬阻断不发出(`route.abort()`)
- ✅ 支路 `dumps`:命中把完整二进制请求体落盘为文件(缺省 mp4);CDP Fetch `postDataEntries` 重组(能抓 File/Blob 上传体,per-page CDP session)
- ✅ 支路 `bodyReplaces`:命中用本地文件字节整体替换请求体(CDP `Fetch.continueRequest`,Content-Length 网络栈重算,复用 dump 的 CDP 会话)
- ✅ 支路 `record`:独立于 `enabled` 的「只记录不改」——拦截所有请求+响应写 `timelines/*.jsonl`(共享 `timeline-recorder.ts`,body 不截断);录制端 CDP Network / 回放端被动 `context.on`
- ✅ 运行中热更新:回放跑到一半改 `request-rules.json` 即时生效(`fs.watchFile`→`updateRequestRules`,route register/unroute 零闲时开销、记录靠常挂监听 + 标志)
- ✅ 响应触发诊断:捕获日志 + 「未命中原因」`explainResponseTriggerMiss`(缺哪个子串/仅空白差异/实际片段/status/bodyJson,signature 去重)

### 待办 / 边界

- 📋 无 UI:规则=编辑 `request-rules.json`(侧栏可视化面板留后续)
- 📋 未覆盖:multipart/form-data、嵌套字段路径、query 改写
- 📋 分端开关(只录制 / 只回放拦截):当前收窄到仅回放;将来加 `applyOnRecord?`/`applyOnReplay?`
- 📋 record MVP:响应只记 status/耗时/mimeType(不记响应体),未覆盖流式/SSE
- 📋 重发边界:页面内 fetch 跨域受 CSP/CORS、需当前有活动页(建议配 `pause` 保活);文件替换峰值内存高(超大优先 `bodyReplaces`);responseTrigger 需 urlPattern 请求先于 triggerUrl 响应被捕获,上传型 body 为空须配 `replaceWithFile`
- 📋 dump/replace 边界:分块/resumable 上传落多个文件需拼接;`replaceWithFile` 绝对路径每次命中读;超大 body 经 CDP 传输有内存开销
