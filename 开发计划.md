# 开发计划

> 状态说明:✅ 已完成 ｜ 🔄 进行中 ｜ 📋 待规划 ｜ ⚠️ 已知问题

本项目为「可视化网页宏录制 + 回放 + 数据提取 + 导出 Excel」桌面工具(MVP 第一版)。
技术栈:Node + TypeScript + Electron + Playwright + exceljs,本地 JSON 存储。

## 模块:内置浏览器与录制

### 核心功能

- ✅ Electron 桌面窗口(地址栏、打开网页、开始/停止录制、运行/保存/加载宏、导出 Excel、日志区)
- ✅ 内置浏览器使用 `<webview>`(Chromium),可浏览任意网页
- ✅ 录制器(`webview-preload.ts`)监听 click / input / press / scroll 并生成步骤
- ✅ 稳定选择器生成(`selector-generator.ts`):data-testid → id → name → aria-label → role+文本 → class → css path → xpath;css path 的每段在可用时携带稳定 class(如 `li.next`),`:nth-of-type` 仅在 class 仍无法区分同标签兄弟时才用,避免「录制后页面增减兄弟元素致选择器命中数变化」
- ✅ click 步骤附语义指纹(`buildFingerprint`:tag/text/aria/href/最近稳定祖先 anchor),随宏 JSON 保存,供回放时主选择器命中 ≠1 通用重定位
- ✅ 录制经 `sendToHost` 回传宿主,渲染层实时累积步骤并展示
- ✅ 导航后自动「重新武装」录制器(保证翻页后操作仍被记录)
- ✅ 新标签页归一:`<webview allowpopups>`(关键前置开关,缺则 Electron 源头丢弃弹窗、handler 不触发)+ 主进程 `did-attach-webview`+`setWindowOpenHandler` 拒绝弹窗并改在同一 webview 内 `loadURL`,录制连续不断、后续操作正常记录;回放端 `context.on('page')` 跟随弹窗切换活动页保持一致
- ✅ 步骤行右键菜单:「标记翻页操作」可将步骤标记为翻页特殊标识并设总页数 N(取消标记/修改页数),标记行带高亮+「翻页 · 共N页」徽标(`pagination`/`pageCount` 字段随宏 JSON 保存)
- ✅ 步骤行右键菜单:「在此后添加滚动到底部」插入 `{type:'scroll-bottom'}` 专用步骤(用于触发无限滚动懒加载),`insertScrollBottom` 仿 `insertPause`;需多次加载就插多条
- ✅ 步骤行右键菜单:「在此后添加等待页面加载完成」插入 `{type:'wait-for-load'}` 专用步骤,`insertWaitForLoad` 仿 `insertScrollBottom`;回放到此等 `load` 事件(整页加载完毕)后再继续
- ✅ 步骤板块 UI 打磨:步骤行 hover 高亮(淡灰底+左侧蓝竖条+平滑过渡),翻页标记行 hover 底色协调加深,翻页徽标升级为渐变 chip(`»` 前缀+阴影+字重),右键菜单项/输入框统一 hover 与 focus 反馈
- ✅ 提取规则面板可折叠、默认收起(`.collapsible.collapsed`,点标题 `classList.toggle('collapsed')` 展开/收起,箭头指示状态):该框多由 AI 生成、用户极少手改,收起省空间并防误操作;AI 生成后保持收起不自动展开
- ✅ 右侧各板块统一可收起:在原有「提取规则/插件/浏览器登录态」基础上,补齐「AI 提取(`#ai-panel`/`#ai-title`)」「录制步骤(`#steps-panel`/`#steps-title`)」两块的 `collapsible` 折叠;通用折叠规则补全各 body(`#extract`/`.browser-body`/`.ai-body`/`.plugin-body`/`#steps`)的 `display:none`(顺带修复插件面板原先点标题不真正收起的遗漏);步骤面板收起时 `.steps-panel.collapsed{flex:0 0 auto;min-height:0}` 不再占弹性高度。点各自标题 `classList.toggle('collapsed')`
- ✅ 步骤行拖拽排序:每个 `.step-line` 设 `draggable`,HTML5 拖拽(dragstart/dragover/drop/dragend)上下拖动调整顺序,拖动时半透明 + 蓝色插入指示线;复用 `steps` 数组 + `renderSteps()` 重绘,排序结果随「保存宏」落盘;与右键菜单并存,纯前端(`renderer.ts` 的 `moveStep`)
- ✅ 主页面左右宽度可拖动调整:`.browser-panel` 与 `.sidebar` 间插入分隔条 `#main-divider`(`col-resize` 光标+hover/拖动高亮),`renderer.ts` 的 `setupMainDivider` 监听 mousedown/mousemove/mouseup 实时改写 `.sidebar` 的 `flex`/`width`(右栏限 [240px, 75%·主区宽]),拖拽期间 `.main.resizing webview{pointer-events:none}` 遮罩防 webview 吞事件;宽度存 `localStorage`(`macro.sidebarWidth`)下次启动恢复
- ✅ 浏览器板块视觉隔离:webview 外包 `.browser-panel`(整圈边框+圆角,`.main` 加 padding/gap 与工具栏/侧栏/日志区拉开间隙),板块头部 `.browser-header` 蓝色左强调条 + 内联 Chromium SVG 图标 + 「Chromium · 内置」名称徽标,标明录制/预览所用浏览器内核

## 模块:回放引擎(MacroRunner)

### 核心功能

- ✅ 读取 JSON 宏,Playwright 启动 Chromium(headed)逐步执行
- ✅ 每种 step 独立处理方法(goto / click / fill / press / scroll / waitForSelector)
- ✅ 每步执行前后打印中文日志(第 N/总 步)
- ✅ 出错截图保存到 `errors/`,返回结构化错误(步骤索引 / 类型 / selector / URL / 原因)
- ✅ 回放全局默认超时(默认 60s,`MACRO_TIMEOUT` 毫秒可覆盖):`page.setDefaultTimeout` / `setDefaultNavigationTimeout`,解决点击后慢页导航等待超时
- ✅ 翻页步骤处理:正常回放跳过被标记的翻页步骤;收集所有翻页步骤构造「翻页序列」(`turnPage` 闭包按文档顺序执行,执行后 `waitForLoadState('domcontentloaded')` 等整页导航稳定再交回提取,防半载页少采/旧分页器误判),总页数取各步 `pageCount` 最大值,传入提取流程驱动翻页
- ✅ click 通用降级:`handleClick` 先 `locator(selector).count()`,唯一命中走主路径;命中 0/多个/非法时若 step 带语义指纹则 `relocateByFingerprint`(anchor 缩小 → 文本精确 → aria → href,取首个可见可用元素)通用重定位;无指纹/重定位失败回退原生 click 沿用出错路径。翻页 click 自动受益,无翻页专用代码
- ✅ 人工介入暂停(`pause` 步骤):回放到该步阻塞,有头浏览器停住等用户在窗口里手动操作(登录/验证码/扫码),点「继续」恢复;依赖倒置——`MacroRunner` 构造注入 `onPause` 回调(core 层不依赖 Electron,无回调默认放行),主进程用 `runId` 隔离的 `resume-macro` 信号恢复;可选 `timeout` 防无人值守永久挂死,超时落入现有出错截图流程;窗口关闭中断回放(`wc.once('destroyed')` reject)
- ✅ 滚动到底部(`scroll-bottom` 步骤):普通顺序步骤,回放到此 `handleScrollBottom` 取页面实际高度 `window.scrollTo(0,document.body.scrollHeight)` 滚到底 + 固定等 1s 懒加载就绪;用于无限滚动页(录不出固定座标),需多次加载就插多条
- ✅ 等待页面加载完成(`wait-for-load` 步骤):普通顺序步骤,回放到此 `handleWaitForLoad` 调 `page.waitForLoadState('load')` 等整页(DOM+所有资源)加载完毕再继续;超时只告警不致命(防轮询型站点 `load` 永不触发挂死),沿用全局默认超时

- ✅ 登录态复用(三条可叠加 UI 开关,默认关,存 `browser-config.json`):①持久化回放 profile——Playwright 改 `launchPersistentContext(userDataDir)`,跨次回放复用同一目录(含 cookie/localStorage),一次登录长期有效;②录制→回放 cookie 注入——回放前从 webview 默认 session 导出 cookies,转 Playwright 格式后 `addCookies` 注入;③录制→回放 localStorage 注入(`injectRecordingLocalStorage`/`#bc-localstorage`)——主进程持有 guest webContents,`executeJavaScript` 读「录制 webview 当前页 origin」的 localStorage,回放端独立 `addInitScript` 按 origin 精准注入(限当前页 origin)。`MacroRunner` 第 4 参 `SessionOptions{userDataDir?,cookies?,localStorage?}`(core 不依赖 Electron,来源/转换在主进程),目录可在侧栏「浏览器登录态」面板勾选/选目录
- ✅ 回放反检测加固(Playwright 自带「Chromium for Testing」易被反爬识别):①加固启动参数(`--disable-blink-features=AutomationControlled` 等 + `ignoreDefaultArgs:['--enable-automation']`);②内核优选回退链(`preferSystemChrome` 开时 `chrome`→`msedge`→捆绑 Chromium 逐个 try/catch 回退);③`addInitScript` 抹 `navigator.webdriver`、补 `window.chrome`/`permissions.query`/`languages`/`plugins`;④仅回退捆绑 Chromium 时规整 UA/locale/viewport。开关 `browser-config.json.useSystemChrome`(默认开)+ UI `#bc-chrome`。后续遇 Cloudflare/DataDome 深度指纹可升级 patchright

## 模块:数据提取与导出

### 核心功能

- ✅ `extract(page, config, pagination?, downloads?)` 支持 single / list / list-detail / list-action 四种模式(list-detail:列表页收基础字段,按 `detailLinkField` 所选字段取详情链接,再逐个进详情页抓详情字段合并成行)
- ✅ list-detail 详情入口改为「用户选字段」:`detailLinkField`(字段名,引用 `fields`)取代 AI 生成的 `detailLinkSelector`;`collectListDetailPage` 用所选字段的 selector 在项内取 href 绝对化(字段缺失/留空→取列表项自身);彻底移除 detailLinkSelector,旧宏需重新生成
- ✅ list-action 模式(列表逐项动作):遍历 `listSelector` 逐项点 `actionSelector`(留空点项本身),每点一项等这次下载开始(`waitForNext`)再点下一项,超时只告警继续;复用翻页上下文逐页处理;不产出数据行,下载结果经 `RunResult.downloads` 回传
- ✅ 通用下载捕获(context 级原语,对所有 mode 生效):回放 context 统一 `acceptDownloads:true`,`core/download-manager.ts` 的 `DownloadManager` 挂 `context.on('download')`,用 `suggestedFilename()` 落盘 `downloads/`,重名加 `(N)` 去重;暴露 `savedPaths`/`count()`/`waitForNext(timeoutMs)`。单个下载按钮等普通场景亦受益
- ✅ 多页采集:list / list-detail 接收翻页上下文(`{ totalPages, turnPage }`),按总页数逐页采集,每页采完执行翻页序列并等待 `listSelector` 就绪;无翻页时按单页处理(向后兼容)
- ✅ list-detail 先收完整列表再进详情:跨所有页把列表项+详情链接全部收集后,才统一逐个进详情页抓取(保证完整列表优先)
- ✅ 字段类型 text / html / attr / href / src,缺失返回空串(不截断数据)
- ✅ `exportToExcel(rows, path)` 首行字段名 + 数据行
- ✅ 导出 Excel 弹出保存对话框可选路径(默认目录 `exports/`、默认名 `result-{时间戳}.xlsx`),导出成功后 `shell.showItemInFolder` 自动打开所在文件夹并高亮该文件;取消导出返回 null 并提示

## 模块:后处理器(回放产出后的定制处理)

> 轻量「type → {manifest,handler}」注册表机制,非通用插件框架;**UI 的可选插件列表由注册表驱动(热插拔)**,为 list-action 下载等场景的定制下游处理提供可插拔扩展点,不动 mode 分发。

### 核心功能

- ✅ 后处理器注册表 `core/post-processors/index.ts`:`registerPostProcessor(manifest, handler)` + `runPostProcessors(specs, ctx)` + `listPostProcessors()`(按序执行、单个失败不致命、core 层不依赖 Electron);`Macro.postProcess?:PostProcessSpec[]`(`{type,options?}`)挂配置,`RunResult.postProcessed?:PostProcessResult[]`(`{type,output?,message}`)回传结果,`PostProcessorManifest`(`{type,label,description}`)驱动 UI
- ✅ 首个后处理器 `merge-zip-excel`(**多格式**):list-action 逐项下载得到一批表格(zip 内的 csv/xls/xlsx,或直接下载的裸表格),解压(`adm-zip`,从 entry 直取 Buffer)→ **SheetJS(`xlsx` 库)通用读** csv/xls/xlsx(取首个 sheet,`sheet_to_json({defval:'',raw:false})`)→ 堆叠合并(union 列、缺列填空,复用现有 `exportToExcel` 写盘)→ 产出 `exports/merged-<时间戳>.xlsx`;**csv 编码 UTF-8 主 + GBK 兜底**(`iconv-lite`,去 BOM);`options.addSourceColumn` 为真加「来源文件」列(默认关,无 UI 入口);只处理本次 `result.downloads`,不扫历史旧文件。SheetJS 装官方已修复版(CDN tarball,避开 npm 0.18.5 安全告警)
- ✅ 主进程串接:`main.ts` 的 `run-macro` 在回放成功且 `macro.postProcess` 非空时 `runPostProcessors`,`shell.showItemInFolder` 优先定位合并产物;IPC `list-plugins`→`listPostProcessors()`、`run-plugin`(弹文件夹选择→对其内文件直接跑插件,不回放),`preload` 暴露 `listPlugins()`/`runPlugin()`
- ✅ UI——**注册表驱动的可选插件列表(热插拔)**:侧栏 collapsible 面板 `#plugin-panel`(青绿 + 🧩 徽标),`#plugin-list` 由 `loadPlugins()` 拉 `listPlugins()` 动态渲染每个插件一行(`.plugin-row`:复选框 + 名称 + **「直接运行」按钮**,下方说明)。**两种触发**:(a) 勾选 → `buildMacro` 写 `macro.postProcess`(随宏保存)→ **随主「运行」执行**,结果走公共 `reportRunResult`,加载宏 `refreshPluginSelection` 回显;(b)「直接运行」`runPluginDirect` → **不跑宏**,经 IPC `run-plugin` 弹**文件多选**(限 zip/csv/xls/xlsx,默认 downloads/),对所选文件直接合并(可直接挑 zip 压缩包)。选择器属提取规则,插件只对文件/下载产物通用
- ✅ 离线自检 `scripts/verify-merge.mjs`(多格式:zip 内 xlsx + UTF-8/BOM csv + 裸 csv + GBK csv,断言合并 6 行、列并集 `[名称,价格,产地,状态]`、各编码中文不乱码,已过);真实样本实测 20 个 payment zip 合并 4506 行通过
