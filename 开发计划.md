# 开发计划

> 状态说明:✅ 已完成 ｜ 🔄 进行中 ｜ 📋 待规划 ｜ ⚠️ 已知问题

本项目为「可视化网页宏录制 + 回放 + 数据提取 + 导出 Excel」桌面工具(MVP 第一版)。
技术栈:Node + TypeScript + Electron + Playwright + exceljs,本地 JSON 存储。

## 模块:内置浏览器与录制

### 核心功能

- ✅ Electron 桌面窗口(地址栏、打开网页、开始/停止录制、运行/保存/加载宏、导出 Excel、日志区)
- ✅ 内置浏览器使用 `<webview>`(Chromium),可浏览任意网页
- ✅ 录制器(`webview-preload.ts`)监听 click / input / press / scroll 并生成步骤
- ✅ 稳定选择器生成(`selector-generator.ts`):data-testid → id → name → aria-label → role+文本 → class → css path → xpath
- ✅ 录制经 `sendToHost` 回传宿主,渲染层实时累积步骤并展示
- ✅ 导航后自动「重新武装」录制器(保证翻页后操作仍被记录)
- ✅ 新标签页归一:点击 target=_blank / window.open 链接时,主进程 `did-attach-webview`+`setWindowOpenHandler` 拒绝弹窗并改在同一 webview 内 `loadURL`,录制连续不断、后续操作正常记录;回放端 `context.on('page')` 跟随弹窗切换活动页保持一致
- ✅ 步骤行右键菜单:「标记翻页操作」可将步骤标记为翻页特殊标识并设总页数 N(取消标记/修改页数),标记行带高亮+「翻页 · 共N页」徽标(`pagination`/`pageCount` 字段随宏 JSON 保存)
- ✅ 步骤板块 UI 打磨:步骤行 hover 高亮(淡灰底+左侧蓝竖条+平滑过渡),翻页标记行 hover 底色协调加深,翻页徽标升级为渐变 chip(`»` 前缀+阴影+字重),右键菜单项/输入框统一 hover 与 focus 反馈
- ✅ 提取规则面板可折叠、默认收起(`.collapsible.collapsed`,点标题 `classList.toggle('collapsed')` 展开/收起,箭头指示状态):该框多由 AI 生成、用户极少手改,收起省空间并防误操作;AI 生成后保持收起不自动展开
- ✅ 步骤行拖拽排序:每个 `.step-line` 设 `draggable`,HTML5 拖拽(dragstart/dragover/drop/dragend)上下拖动调整顺序,拖动时半透明 + 蓝色插入指示线;复用 `steps` 数组 + `renderSteps()` 重绘,排序结果随「保存宏」落盘;与右键菜单并存,纯前端(`renderer.ts` 的 `moveStep`)
- ✅ 浏览器板块视觉隔离:webview 外包 `.browser-panel`(整圈边框+圆角,`.main` 加 padding/gap 与工具栏/侧栏/日志区拉开间隙),板块头部 `.browser-header` 蓝色左强调条 + 内联 Chromium SVG 图标 + 「Chromium · 内置」名称徽标,标明录制/预览所用浏览器内核

## 模块:回放引擎(MacroRunner)

### 核心功能

- ✅ 读取 JSON 宏,Playwright 启动 Chromium(headed)逐步执行
- ✅ 每种 step 独立处理方法(goto / click / fill / press / scroll / waitForSelector)
- ✅ 每步执行前后打印中文日志(第 N/总 步)
- ✅ 出错截图保存到 `errors/`,返回结构化错误(步骤索引 / 类型 / selector / URL / 原因)
- ✅ 回放全局默认超时(默认 60s,`MACRO_TIMEOUT` 毫秒可覆盖):`page.setDefaultTimeout` / `setDefaultNavigationTimeout`,解决点击后慢页导航等待超时
- ✅ 翻页步骤处理:正常回放跳过被标记的翻页步骤;收集所有翻页步骤构造「翻页序列」(`turnPage` 闭包按文档顺序执行),总页数取各步 `pageCount` 最大值,传入提取流程驱动翻页
- ✅ 人工介入暂停(`pause` 步骤):回放到该步阻塞,有头浏览器停住等用户在窗口里手动操作(登录/验证码/扫码),点「继续」恢复;依赖倒置——`MacroRunner` 构造注入 `onPause` 回调(core 层不依赖 Electron,无回调默认放行),主进程用 `runId` 隔离的 `resume-macro` 信号恢复;可选 `timeout` 防无人值守永久挂死,超时落入现有出错截图流程;窗口关闭中断回放(`wc.once('destroyed')` reject)

- ✅ 登录态复用(两条可叠加 UI 开关,默认关,存 `browser-config.json`):①持久化回放 profile——Playwright 改 `launchPersistentContext(userDataDir)`,跨次回放复用同一目录(含 cookie/localStorage),一次登录长期有效;②录制→回放 cookie 注入——回放前从 webview 默认 session 导出 cookies,转 Playwright 格式后 `addCookies` 注入。`MacroRunner` 第 4 参 `SessionOptions`(core 不依赖 Electron,cookie 来源/转换在主进程),目录可在侧栏「浏览器登录态」面板勾选/选目录

### 待办

- 📋 步骤级超时与重试策略可配置
- 📋 localStorage 跨录制(Electron)→回放(Playwright)注入(cookie 已支持,localStorage 暂靠持久 profile)

## 模块:数据提取与导出

### 核心功能

- ✅ `extract(page, config, pagination?)` 支持 single / list / list-detail 三种模式(list-detail:列表页收基础字段+详情链接,再逐个进详情页抓详情字段合并成行)
- ✅ 多页采集:list / list-detail 接收翻页上下文(`{ totalPages, turnPage }`),按总页数逐页采集,每页采完执行翻页序列并等待 `listSelector` 就绪;无翻页时按单页处理(向后兼容)
- ✅ list-detail 先收完整列表再进详情:跨所有页把列表项+详情链接全部收集后,才统一逐个进详情页抓取(保证完整列表优先)
- ✅ 字段类型 text / html / attr / href / src,缺失返回空串(不截断数据)
- ✅ `exportToExcel(rows, path)` 首行字段名 + 数据行
- ✅ 导出 Excel 弹出保存对话框可选路径(默认目录 `exports/`、默认名 `result-{时间戳}.xlsx`),导出成功后 `shell.showItemInFolder` 自动打开所在文件夹并高亮该文件;取消导出返回 null 并提示

### 待办

- 📋 自动识别列表结构(免手动配置)
- 📋 CSV 导出

- ✅ HTML 发送前降噪(去 script/style/注释,不截断长度);从回复剥出 JSON(处理 ```json 围栏),解析为 ExtractConfig
- ✅ 配置文件 `ai-config.json`(项目根,首次运行自动生成);系统/提示词模板、超时、连接覆盖可配
- ✅ 自检脚本 `scripts/test-ai.mjs`(已实跑:对接 webextract,24s 产出合法 list 规则)
- ✅ IPC `ai-list-profiles` / `ai-generate-extract` + preload 暴露 + UI「AI 提取」区(agent 下拉 + 需求输入 + 生成按钮)
- ✅ 渲染层:抓当前 `<webview>` 页面 HTML → 调 openclaw agent → 规则回填「提取规则」框
- ✅ 目标 mode 选择:「AI 生成规则」旁加 mode 下拉(single/list/list-detail),所选模式作为「前提」注入提示词;list-detail 需「提取规则」框已有合法 list 规则,并把该 list 规则带给 agent 作基础增量补 detail;SOUL.md 已补 list-detail 结构定义

- ✅ AI 板块 UI 打磨:面板标题紫色强调条+圆点,引擎/模式下拉与需求框统一 focus 紫光环,生成按钮按下下沉+阴影过渡,生成中状态区显示旋转加载圆环(`setAiBusy` 切换 `ai-loading` class),生成结果状态文字成功转绿/失败转红(`ok`/`err` class)
- ✅ 上传 `ai-config.json`:AI 面板「上传配置」按钮,主进程弹文件框选 json → core `validateAiConfig` 严格校验格式(profiles 非空/各字段类型/id 唯一/defaultProfile 一致/openclaw.identity 齐全/cleanHtml 布尔等,失败返回中文原因)→ 通过才 `importAiConfig` 覆盖生效(覆盖前备份 `ai-config.json.bak`),即时刷新引擎下拉;校验失败不覆盖原文件。链路 core→main(`import-ai-config` IPC)→preload→renderer
