拦截器规则配置手册

request-rules.json

项目根 /（打包态 userData）下的请求拦截配置。本手册整理出所有可配置字段、作用与限制。改动文件后约 1 秒内热更新生效,无需重启;回放跑到一半改也立即生效。

§顶层结构总览

配置根对象:8 个支路键 + 顶层 maxResendHops(响应触发连环的跳数上限)。除 record 外,所有支路都受顶层 enabled 总开关统管。

按条件改写响应头

§通用概念（先读这里，能省一半试错）

① CDP glob 匹配（所有 urlPattern / triggerUrl）

URL 匹配用 CDP glob 方言(不是正则、不是 Playwright glob),对整条 URL 全串匹配:

写法含义示例
*匹配任意长度任意字符*/api/search* 命中含 /api/search 的任何 URL
?匹配单个字符/item/?
其它字符字面量(正则元字符会被自动转义)*upload.youtube.com*
易踩坑模式是对完整 URL 匹配。想匹配 /api/x 通常要写成 */api/x*(前后各加 *),否则协议 / 域名 / 查询串会让它匹配不上。

② bodyType 嗅探优先级

rules / resends 的 bodyType 缺省时,按此顺序判定:

规则显式 bodyType  >  请求 Content-Type 头  >  body 首字符是 { 判 json  >  兜底 form

③ set / append / remove 语义（rules 与 resends 共用）

只作用于 body 顶层字段不支持嵌套点路径(如 a.b.c)。同一条规则内执行顺序固定:set → append → remove。
动作json bodyform body
set整体覆盖字段值,保留原始 JSON 类型(数组 / 对象 / 数字照写照留)转为字符串写入
append往数组字段追加元素并去重;字段不存在则新建数组、原为单值则包成数组。值写要追加的元素数组追加为重复参数(tags=a&tags=b)、同值去重
remove删除列出的顶层字段名(字符串数组)
数组值写法数组值直接写真 JSON 数组 ["x"],不要写成字符串 "['x']"(带引号 / 单引号非合法 JSON)。这是最常见的配置错误。

④ 为什么支路是分开的数组

匹配引擎「首个命中即返回」。若把改写 / 重发 / 阻断混进同一个数组,会互抢首命中。因此 rules / resends / responseRules / blocks / dumps / bodyReplaces 各自独立数组,互不干扰,同一个 URL 可同时命中多个支路。

⑤ 防递归标记头 + 链式跳数熔断

重发的请求会带 x-macro-resend 头,其值 = 该请求在重发链上的跳数(hop):真实浏览器请求不带此头(第 0 跳),第一次重发带 1,之后每被链式重发一次 +1。route 改写 / dumps 落盘 / 请求捕获三处见到此头即整体跳过(不再加工工具自己发的请求)。该标记头不可被 setHeaders 覆盖、不可被 removeHeaders 删除。

响应触发支持连环:一条重发的响应可以继续触发下一条 responseTrigger 规则(A→B→C…),用 setHeaders 盖业务标记头 + 下一条规则的 requestHeaders 条件精确接棒即可串起来。为防无限自环/互环,当触发响应所属请求的跳数达到 maxResendHops(顶层配置,缺省 5)即熔断,不再继续触发并打一条日志。想串更长的链就调大它;只想单发不连环就设成 1。

⑥ Windows 路径转义

JSON 里反斜杠要转义:"D:\\videos\\new.mp4"。replaceWithFile 必须是绝对路径。

⑦ 归一化 = 静默丢弃非法项

加载时逐条校验:缺 urlPattern 的规则、缺 replaceWithFile 的 bodyReplaces、缺 triggerUrl 的 responseTrigger 会被整条丢弃(不报错,只是不生效)。坏 JSON 则整份回退为空配置。配了没反应先检查这几处。

1rules[] 请求体改写

命中 urlPattern 的 POST 请求,就地改写其 body 顶层字段。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配请求 URL缺失整条丢弃
bodyType"json" | "form"可选指定 body 类型省略则自动嗅探(见通用概念②)
setobject可选设置 / 覆盖顶层字段仅顶层;数组值写真数组
appendobject可选往顶层数组字段追加去重仅顶层
removestring[]可选删除顶层字段仅顶层
限制
  • 仅回放端;仅对 POST 请求生效。
  • 只改 body 顶层字段,不支持嵌套点路径。
  • 未覆盖 multipart/form-data;非法 JSON 原样放行不改。
  • 要「就地替换整个 body 成一个文件」用 bodyReplaces,不是这里。
// 覆盖 pageSize、把 videoIds 整体设成数组、删 debugFlag
{
  "urlPattern": "*/api/search*",
  "bodyType": "json",
  "set": { "pageSize": 100, "videoIds": ["B5IvLw_ihbU"] },
  "remove": ["debugFlag"]
}

2resends[] 重发型

命中 urlPattern 的请求作为触发器,延时后取原 body 改参、主动重新发起一个新请求(不改原请求,原请求照常放行)。这是本模块唯一会「主动发 HTTP 请求」的支路。

仅回放端受 enabled 管

通用字段

字段类型必填作用限制 / 备注
urlPatternstring必填触发观察的 URL(请求触发=命中即重发它;响应触发=要捕获并重发的请求)缺失整条丢弃
delayMsnumber可选命中后首次重发延时(毫秒)。间隔 n 秒 = n×1000缺省 0=立即;负数被忽略
method"POST" | "GET"可选重发使用的方法缺省 POST。仅请求触发生效;响应触发用捕获请求的原方法
bodyType"json" | "form"可选body 类型省略按触发请求嗅探
set / append / removeobject / object / string[]可选对重发副本改哪些参数(语义同 rules)都不填=原样重发;设了 replaceWithFile 则全被忽略;响应触发下值里可用 {{变量}} 注入 responseTrigger.extract 提取的响应值
replaceWithFilestring(绝对路径)可选整体用该文件完整字节作重发 body(能发 mp4 等任意二进制)忽略 set/append/remove/bodyType;content-type 保留触发请求原值;读文件失败则跳过本次重发
repeatnumber可选一次触发重发几次缺省 1,clamp 到 [1,100]
intervalMsnumber可选repeat>1 时相邻两次的间隔(毫秒)缺省 0
dedupeMsnumber可选去抖窗口:N 毫秒内同规则只发一次(防高频触发放大)缺省 0=每次命中都重发
setHeadersobject(string→string)可选设置 / 覆盖重发请求头(换 token 等)大小写不敏感覆盖;x-macro-resend 不可覆盖;响应触发下值里可用 {{变量}} 注入 responseTrigger.extract 提取的响应值
setUrlstring(URL 模板)可选重发目标 URL:设了则整体覆盖捕获请求的原 URL(如把新 id 拼进重发地址)响应触发下值里可用 {{变量}} 注入 responseTrigger.extract 提取的响应值;渲染后为空/纯空白则回退原 URL;只改 URL,method/体不变;无变量时=静态改端点
removeHeadersstring[]可选删除重发请求头(可删继承自触发请求的头)大小写不敏感;x-macro-resend 不可删
responseTriggerobject可选设了则改由响应触发(见下表)不设=请求侧命中即触发

responseTrigger 子对象 —— 响应条件触发（捕获-重放模型）

语义:捕获命中顶层 urlPattern 的请求(记下 url/method/头/体),当 triggerUrl 的响应满足以下条件(全部 AND)时,重发捕获的那个请求。即「监听 triggerUrl 的响应 → 重发 urlPattern 捕获的请求」。

字段类型必填作用限制 / 备注
triggerUrlstring必填监听哪个响应作触发闸门(CDP glob 匹配响应 URL)缺失则整条 resends 规则被丢弃
statusnumber可选响应状态码需等于此值如 200
headersobject(string→string)可选响应头需全部相等(AND)头名大小写不敏感,值精确相等
requestHeadersobject(string→string)可选请求头条件:triggerUrl 那条请求的头需全部相等(AND)头名大小写不敏感,值精确相等;与 headers(响应头)对称,判同一 triggerUrl 事务的请求侧;同步可得,不触发读体
bodyJsonobject(点路径→值)可选响应体 JSON 按点路径取值,全部相等(AND)JSON.parse 后 a.b.c 逐层取,String() 后精确比;解析失败/路径缺失/读不到体=不命中
bodyContainsstring[]可选响应体原文子串需全部出现(AND)不解析 JSON、不依赖路径,适配深层嵌套/异构数组;大小写敏感;中文若被 \u 转义会失配,优先用 ASCII 标记
extractobject(变量名→取值源)可选从触发响应提取命名变量,供本规则 set/append/setHeaders/setUrl 用 {{name}} 占位注入重发(如把响应里的新 token 写进重发请求头、或把新 id 拼进重发 URL)每个源二选一:fromBody(响应体 JSON 点路径,复用 bodyJson 的取值方式;命中对象/数组→JSON.stringify)或 fromHeader(响应头名,大小写不敏感);取不到用 default(缺省空串)。有 fromBody 会异步读响应体
各组条件都可选status / headers / requestHeaders / bodyJson / bodyContains 都不给,则该响应恒满足条件(命中即触发)。只要有 bodyJson / bodyContains、或 extract 里有 fromBody 源,才会异步读响应体(status / headers / requestHeaders 先筛,均同步)。
extract → 占位符注入触发命中后,extract 从触发响应取值得到命名变量;在动作字段 set/append(body 参数)/setHeaders(请求头)/setUrl(重发 URL)的值里写 {{变量名}} 即被替换为提取值(set/append 支持嵌套对象/数组的字符串叶子)。占位符仅在规则的动作值里生效,不影响真实数据;未知变量渲染为空串。请求头仅支持 set 注入(setHeaders),无 append;setUrl 是整条 URL 模板,渲染后为空则回退原 URL。
resends 限制
  • 仅回放端。重发本身是页面内 fetch(带页面 cookie/登录态,DevTools Network 可见),需当前有活动页且宏此刻仍在跑(响应触发场景常配 pause 步保活),否则重发被跳过。
  • 跨域重发受页面 CSP / CORS 限制。
  • 响应触发:urlPattern 请求须先于 triggerUrl 响应被捕获,否则跳过+日志;捕获只取最近一次。
  • 捕获上传型(File/Blob)请求的 body 会为空(被动读 Blob 返回 null),此类要重发完整字节须配 replaceWithFile(url/头仍能捕获)。
  • 响应体读取走 Playwright 网络层,不受页面 CORS 限制(能读跨域响应体);但重发的 fetch 仍受 CORS 限。
  • 触发限于 POST/GET;超大文件重发内存开销比 bodyReplaces 大,超大上传优先用 bodyReplaces。
// 响应触发 + bodyContains 免路径匹配(YouTube 上传完成后重发 createvideo)
{
  "urlPattern": "*v1/upload/createvideo*",        // 捕获这个请求
  "responseTrigger": {
    "triggerUrl": "*/upload/feedback*",          // 监听这个响应
    "status": 200,
    "bodyContains": ["uploadFeedbackItemContinuation", "\"fractionCompleted\":1"]
  },
  "delayMs": 800,
  "repeat": 1
}
// 响应触发 + requestHeaders 请求头门控:仅当 triggerUrl 那条请求带 x-goog-upload-command:upload,finalize 时才重发
{
  "urlPattern": "*v1/upload/createvideo*",        // 捕获这个请求
  "responseTrigger": {
    "triggerUrl": "*/upload/feedback*",          // 监听这个响应
    "status": 200,
    "requestHeaders": { "x-goog-upload-command": "upload, finalize" }  // triggerUrl 请求头需相等
  },
  "delayMs": 500,
  "repeat": 1
}
// 响应触发 + extract:从触发响应取值,经 {{占位符}} 注入重发的请求头 / 请求体参数 / 重发 URL
{
  "urlPattern": "*/api/submit*",                 // 捕获这个请求
  "responseTrigger": {
    "triggerUrl": "*/api/status*",             // 监听这个响应
    "bodyJson": { "data.state": "done" },     // 门控:达标才重发
    "extract": {                                // 从触发响应提取命名变量
      "token": { "fromBody": "data.uploadToken" },     // 响应体点路径取值
      "rid":   { "fromBody": "data.resourceId" },       // 供拼进重发 URL
      "sid":   { "fromHeader": "x-goog-session-id", "default": "" }  // 响应头取值
    }
  },
  "setHeaders": { "authorization": "Bearer {{token}}", "x-sid": "{{sid}}" },  // 注入请求头
  "set": { "session_id": "{{sid}}" },       // 注入请求体参数
  "setUrl": "https://host/api/v2/{{rid}}/commit?token={{token}}"  // 整体覆盖重发 URL,注入提取值
}

3responseRules[] 响应头条件改写

命中 urlPattern 的响应,当其响应头满足 when 条件时,改写响应头。条件读的是响应头、动作改的也是响应头。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配响应对应的请求 URL与 method 无关,GET 也能改;缺失整条丢弃
whenobject(string→string)可选这些响应头需全部相等才改(AND)头名大小写不敏感;缺省=无条件总是改
setHeadersobject(string→string)可选设置 / 覆盖响应头大小写不敏感覆盖,不产生重复键
removeHeadersstring[]可选删除响应头大小写不敏感
限制命中的请求由浏览器直连改为 Node 侧 route.fetch() 代发(有开销,只作用命中 URL,其余请求零成本);多个合法重复头(如多个 set-cookie)中不在名单内的会被保留。
// 响应头 xx=1 时,把 cc 设为 1 并删掉 x-drop 头
{
  "urlPattern": "*/api/data*",
  "when": { "xx": "1" },
  "setHeaders": { "cc": "1" },
  "removeHeaders": ["x-drop"]
}

4requestHeaderRules[] 请求头条件改写

命中 urlPattern 的请求,当其原始请求头满足 when 条件时,在发出前改写请求头。对称于 responseRules,但改的是请求侧。典型用途:注入/替换 Authorization、改 X-Requested-With、删探针头。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配请求 URL与 method 无关,GET 也能改;缺失整条丢弃
whenobject(string→string)可选这些请求头需全部相等才改(AND)头名大小写不敏感;缺省=无条件总是改;读的是改写前原始请求头,不含 cookie
setHeadersobject(string→string)可选设置 / 覆盖请求头大小写不敏感覆盖,不产生重复键
removeHeadersstring[]可选删除请求头大小写不敏感;cookie/host 无法删
限制
  • 仅回放端。改的是原始在途请求本身的头(区别于 resends 的 setHeaders——那改的是重发副本的头)。
  • cookie / host 改不了:由浏览器管理,route.continue 无法覆盖/删除;request.headers() 本就不含 cookie,浏览器会自动补回(不误删登录态)。改 cookie 用登录态注入 / 持久 profile。
  • content-length 会被自动剥离:同一请求的 body 可能被 rules 一并改写,交网络栈按真实 body 重算,避免长度不符。
  • 勿用 removeHeaders 清空全部头。
// 请求头 x-flag=1 时,换 Authorization、删继承的 x-trace;不满足则原样放行
{
  "urlPattern": "*/api/order*",
  "when": { "x-flag": "1" },
  "setHeaders": { "authorization": "Bearer NEW_TOKEN" },
  "removeHeaders": ["x-trace"]
}

5blocks[] 真拦截（硬阻断）

命中 urlPattern 的请求直接阻断、不让其发出(route.abort(),页面的 fetch/XHR 收到网络错误)。这是本模块唯一「不放行」的分支。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配请求 URL缺失整条丢弃
methodstring可选只拦指定 HTTP 方法(如 POST/GET)大小写不敏感;缺省=拦该 URL 所有方法
 
// 屏蔽埋点上报;下例只拦 POST,GET 照常放行
{ "urlPattern": "*/api/track*" }
{ "urlPattern": "*/api/report*", "method": "POST" }

6dumps[] 请求体落盘

命中 urlPattern 的请求,把其完整二进制请求体(从第一字节到最后一字节)写成一个文件到 dumps/ 目录。被动观察、不改原请求。用于抓上传型接口的字节体(如把视频上传体存成 mp4)。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配请求 URL缺失整条丢弃
methodstring可选只落指定方法(如 PUT/POST)大小写不敏感;缺省=落所有方法(上传常是 PUT/POST)
extensionstring可选输出文件后缀可含或不含前导点(mp4 或 .bin);缺省 mp4
走 CDP Fetch,能抓 File/Blob 上传体用底层 CDP postDataEntries 逐字节重组,能抓到 Playwright 高层 API 拿不到的 File/Blob 上传体。每命中一个请求落一个文件,文件名 dump-<毫秒戳>-<序号>.<后缀>。
限制分块 / resumable 上传会落多个文件需自行拼接;个别未缓冲的流式上传可能拿不到则静默跳过。

7bodyReplaces[] 请求体整体替换

命中 urlPattern 的请求,在拦截点把其整个请求体替换成本地文件的完整字节,再放行发出。与 dump 是一对(dump 读、这个写)。走 CDP,能替换 File/Blob 上传体(如把上传的视频换成本地另一个 mp4)。

仅回放端受 enabled 管
字段类型必填作用限制 / 备注
urlPatternstring必填CDP glob 匹配请求 URL缺失整条丢弃
methodstring可选只替换指定方法(如 PUT/POST)大小写不敏感;缺省=替换所有方法
replaceWithFilestring(绝对路径)必填用其完整字节整体替换请求体缺失整条丢弃;读文件失败则原样放行不替换;Windows 路径反斜杠转义
Content-Length 自动重算替换成不同大小的文件,网络栈会自动重算 Content-Length,无需手动处理。
// 把 YouTube 上传(PUT)的请求体整个换成本地另一个 mp4 再发出
{
  "urlPattern": "*upload.youtube.com*",
  "method": "PUT",
  "replaceWithFile": "D:\\videos\\replacement.mp4"
}

8record 只记录不修改

拦截所有请求(不限 method)+ 响应,写入 timelines/timeline-*.jsonl 供事后分析。不改写任何请求。

录制端 + 回放端独立开关,与 enabled 无关
字段类型必填作用限制 / 备注
enabledboolean必填true 即开启记录与顶层 enabled 无关,即便改写关闭也照记
urlPatternstring可选只记录命中的 URL缺省 / * = 记录所有
includeBodyboolean可选是否记录完整请求 body缺省 true(禁止截断);false 则不记 body
限制响应只记 status / 耗时 / mimeType,不记响应体;流式 / SSE 若不发 loadingFinished 则无响应行。文件名 timeline-<record|replay>-<毫秒戳>.jsonl,每行一条 JSON,含 url/method/headers/请求body/status/耗时。
与 enabled 各自独立可只开 record(enabled:false)纯记录、也可两者同开(改写命中的 POST + 记录所有请求,记录到的是改写前原始 body)。

§完整示例（可整份拷贝改用）

完整带注释的示例见项目 examples/request-rules.example.json。下面是一份最小可用骨架:

{
  "enabled": true,

  "rules": [
    { "urlPattern": "*/api/search*", "bodyType": "json",
      "set": { "pageSize": 100 } }
  ],

  "resends": [
    { "urlPattern": "*/api/refresh*", "delayMs": 3000,
      "set": { "page": 2 }, "repeat": 1 }
  ],

  "responseRules": [
    { "urlPattern": "*/api/open*",
      "setHeaders": { "access-control-allow-origin": "*" } }
  ],

  "requestHeaderRules": [
    { "urlPattern": "*/api/order*", "when": { "x-flag": "1" },
      "setHeaders": { "authorization": "Bearer NEW_TOKEN" } }
  ],

  "blocks": [
    { "urlPattern": "*/api/track*" }
  ],

  "dumps": [
    { "urlPattern": "*upload.youtube.com*", "method": "PUT", "extension": "mp4" }
  ],

  "bodyReplaces": [
    { "urlPattern": "*upload.youtube.com*",
      "replaceWithFile": "D:\\videos\\new.mp4" }
  ],

  "record": { "enabled": false, "urlPattern": "*", "includeBody": true }
}
配了没反应?排查清单
  • 顶层 enabled 是否为 true?(record 除外)
  • urlPattern 前后是否各加了 *?(对完整 URL 匹配)
  • 数组值是不是写成了字符串 "['x']"?应写真数组 ["x"]。
  • 是否在录制阶段期待改写生效?现只在回放阶段生效(record 除外)。
  • bodyReplaces 缺 replaceWithFile / responseTrigger 缺 triggerUrl → 整条被静默丢弃。
  • JSON 是否合法?坏 JSON 会让整份配置回退为空。
