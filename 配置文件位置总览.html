二爪鱼(网页宏录制/回放/提取/导出工具)全部运行时配置的位置、作用与加载约定 · 收尾于配置统一收拢到 config/ 之后

💡 普通用户不必手改 JSON:侧栏「⚙️ 配置中心」板块可视化编辑运行时配置 —— bank-integrate.json 提供字段级表单(exe 路径/面板描述/示例文件名),其余复杂配置提供「📄 打开文件」+「📂 打开配置文件夹」入口。

目录布局

# dataRoot：开发态 = 项目根;打包态 = %APPDATA%\macro-recorder(由 app.isPackaged 分叉) dataRoot/ ├── config/ # ← 全部 7 个运行时配置统一收于此(首次运行自动生成,已 gitignore) │ ├── ai-config.json # AI 对接 profile / 凭据 │ ├── browser-config.json # 登录态复用 / 反检测开关 │ ├── request-rules.json # 请求拦截 8 支路 │ ├── bank-integrate.json # 银行整合 5 代号 exe 路径 │ ├── merge-config.json # 表格合并解析规则 │ ├── replay-profile.json # 回放行为档(多档可切) │ └── hooks.json # 事件钩子 ├── macros/ # 宏文件 + <宏名>.captures.json 选择器旁车(随宏走,不在 config/) ├── exports/ errors/ downloads/ timelines/ dumps/ # 运行时输出目录 └── browser-profile/ # 持久化浏览器 profile(浏览器状态目录,非配置文件) # 注:config-templates/(git 追踪)是打包预置模板,≠ 运行时 config/;bank-integrate 首次从它拷平台模板。

加载约定(单一事实来源)

  • 位置:主进程 configDir = path.join(dataRoot, 'config') 是唯一事实源,4 个主进程配置路径由它派生;core 层 ai-config 经 MACRO_DATA_DIR/config/ 对齐;后处理器经 ctx.configDir。
  • 骨架:新配置统一走 core/json-config.ts 的 loadJsonConfig(existsSync 不存在 → 建目录写模板 → 存在 → readFileSync + normalize → 坏 JSON 回退现状,永不抛)。
  • 自建目录:各 loader 首次写模板前 mkdirSync(config/),且 ensureDirs() 启动即建,双保险。
  • 模板:值平台无关者走代码内联模板;需随包者落 config-templates/<name>.json + package.json build.extraResources。
  • 忽略:.gitignore 忽略整个 /config/(及根目录旧位置遗留文件),不追踪任何运行时配置。
迁移说明:本次仅调整新生成配置的落点(旧位置在 dataRoot 根),不含旧配置自动迁移逻辑(按约定「不管已安装用户」)。升级后旧根位置的配置文件将不再被读取,程序会在 config/ 下按模板重新生成默认配置;如需保留旧配置,手动把根目录对应 *.json 移入 config/ 即可。
